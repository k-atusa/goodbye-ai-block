# 개인정보 처리방침

**goodbye-ai-block은 웹페이지 내의 난독화된 텍스트와 이미지를 자동으로 복호화하여, 자동화된 검열 및 필터링을 우회하고 AI 크롤러의 무단 스크랩을 방지하면서 사용자가 원본 콘텐츠를 안전하게 감상할 수 있도록 돕는 확장 프로그램입니다.**

## 1. 개인정보 수집 및 전송
본 확장 프로그램은 사용자의 어떠한 개인정보, 브라우징 방문 이력, 웹페이지 내 텍스트 및 이미지 데이터를 외부 서버로 수집, 저장, 전송하지 않습니다. 모든 데이터는 사용자의 기기 내부에서만 처리됩니다.

## 2. 권한 요구 사항 및 목적
본 확장 프로그램은 원활한 기능 작동을 위해 아래의 권한만을 요구하며, 명시된 목적 외에는 절대 사용되지 않습니다.

- **storage**: 사용자가 설정한 Seed(비밀키) 값 및 확장 프로그램의 활성화 상태 옵션을 브라우저 로컬 저장소에 안전하게 보관하기 위해 사용됩니다.
- **activeTab**: 현재 사용자가 보고 있는 탭의 콘텐츠에 접근하기 위해 사용됩니다.
- **host_permissions (<all_urls>)**: 웹페이지 내에 난독화된 텍스트 및 이미지 엘리먼트가 존재하는지 실시간으로 감지하고, 이를 로컬 단에서 복호화하여 화면에 정상적으로 출력하기 위해 필수적으로 사용됩니다.

## 3. 문의 사항
본 프로그램의 데이터 처리 방식이나 권한에 대해 문의 사항이 있으신 경우, 본 깃허브 레포지토리의 이슈(Issues) 탭을 통해 문의해 주시기 바랍니다.

# Privacy Policy

**goodbye-ai-block is a browser extension that automatically detects and deobfuscates encrypted text and images, bypassing automated censorship and filtering while protecting content from unauthorized AI scraping, allowing users to view the original content seamlessly.**

## 1. Data Collection and Transmission
This extension does not collect, store, or transmit any personal information, browsing history, or text/image data found on web pages to external servers. All data processing is performed strictly within the user's local device.

## 2. Requested Permissions and Purpose
This extension requests only the minimum necessary permissions required for its functionality, and they are never used for any purpose other than stated below:

- **storage**: Used to securely save user configuration options, such as the encryption Seed (secret key) and extension enabled status, within the local browser storage.
- **activeTab**: Used to access the content of the currently active tab when interacting with the extension.
- **host_permissions (<all_urls>)**: Essential for real-time detection of obfuscated text and image elements across websites, and deobfuscating them locally to restore original content on the screen.

## 3. Contact & Support
If you have any questions or suggestions regarding this privacy policy or data handling practices, please open an issue in the official GitHub repository's Issues tab.
